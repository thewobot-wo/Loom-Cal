<acceptance_criteria>

## AC-1: Event notifications scheduled with configurable lead time
```gherkin
Given events exist in Convex
When the CalendarViewModel receives subscription updates
Then a local notification is scheduled for each upcoming event at (start - leadMinutes)
And the notification title is the event title
And the notification body shows the event time
And only events in the next 48 hours are scheduled (UNUserNotificationCenter 64-item limit)
```

## AC-2: Task due date notifications scheduled
```gherkin
Given tasks with due dates exist in Convex
When the TaskViewModel receives subscription updates
Then a local notification is scheduled for each incomplete task with a future due date
And tasks with hasDueTime=true notify at the exact due time
And tasks with hasDueTime=false notify at 9:00 AM on the due date
And completed tasks do not generate notifications
```

## AC-3: Lead time is user-configurable
```gherkin
Given the app is running
When the user taps the bell icon in the calendar toolbar
Then a menu shows lead time options (5, 10, 15, 30, 60 minutes)
And the currently selected lead time has a checkmark
And changing the selection persists via UserDefaults
And notifications are rescheduled with the new lead time
```

## AC-4: Notification permission requested on launch
```gherkin
Given the app launches for the first time (or permission not yet determined)
When the app scene appears
Then UNUserNotificationCenter requests alert + sound + badge authorization
And if denied, notifications silently skip scheduling (no crashes)
```

<task type="auto">
  <name>Task 1: Create NotificationService scheduling engine</name>
  <files>LoomCal/Services/NotificationService.swift</files>
  <action>
    Create a new file `LoomCal/Services/NotificationService.swift`:

    1. **Singleton pattern** — `static let shared = NotificationService()` with private init. NOT an ObservableObject (matches NLParseService pattern).

    2. **UserDefaults lead time** — `eventLeadMinutes` property backed by UserDefaults key `"notificationLeadMinutes"` with default 15. Provide a setter that also triggers rescheduling.

    3. **Permission request** — `requestPermission()` async method calling `UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .sound, .badge])`. Log result, do not throw on denial.

    4. **rescheduleEventNotifications(_ events: [LoomEvent])** method:
       - Remove all pending notifications with identifiers prefixed `"event-"`
       - Filter to non-all-day events with start in the next 48 hours AND start - leadMinutes > now
       - For each, create UNMutableNotificationContent (title: event title, body: formatted time string like "in 15 minutes", sound: .default)
       - Create UNCalendarNotificationTrigger from (event start - leadMinutes) date components (year, month, day, hour, minute)
       - Identifier: `"event-\(event._id)"`
       - Add via UNUserNotificationCenter.current().add()

    5. **rescheduleTaskNotifications(_ tasks: [LoomTask])** method:
       - Remove all pending notifications with identifiers prefixed `"task-"`
       - Filter to incomplete tasks with dueDate in the future
       - For hasDueTime=true: trigger at the dueDate time
       - For hasDueTime=false: trigger at 9:00 AM on the due date (user's local timezone)
       - Content: title = task title, body = "Due today" or "Due at [time]", sound: .default
       - Identifier: `"task-\(task._id)"`
       - Add via UNUserNotificationCenter.current().add()

    6. **Helper** — private `removeNotifications(withPrefix:)` method that calls `UNUserNotificationCenter.current().getPendingNotificationRequests()`, filters by prefix, and removes matching identifiers.

    **Avoid:**
    - Do NOT make this an ObservableObject — it's a utility singleton
    - Do NOT import ConvexMobile — this file only needs UserNotifications and Foundation
    - Do NOT schedule notifications for past events/tasks (filter them out)
  </action>
  <verify>File compiles — check syntax is valid Swift with UserNotifications import</verify>
  <done>AC-1 scheduling logic, AC-2 scheduling logic, AC-4 permission method created</done>
</task>
