<objective>
## Goal
Replace the macOS TabView with a NavigationSplitView sidebar layout, so Loom Cal feels native on Mac (sidebar navigation) while iOS retains its native tab bar.

## Purpose
Phase 8 success criteria #1 and #2: iPhone uses a native tab bar, Mac uses NavigationSplitView sidebar with no iOS-specific UI artifacts. The iOS TabView already provides a native bottom tab bar — the main work is giving macOS its own sidebar-based navigation.

## Output
- ContentView.swift refactored with `#if os(macOS)` / `#else` branching at the navigation container level
- macOS: NavigationSplitView with sidebar (Calendar, Tasks, Loom) + detail pane
- iOS: TabView unchanged (already native tab bar)
</objective>

  <action>
    1. Add an `AppSection` enum (calendar, tasks, chat) above ContentView, conforming to Hashable.

    2. Inside ContentView.body, wrap the entire navigation in `#if os(macOS)` / `#else`:

    **macOS branch — NavigationSplitView:**
    - Use `@State private var selectedSection: AppSection = .calendar`
    - NavigationSplitView with sidebar containing a `List(selection: $selectedSection)` with three `Label` rows tagged to AppSection cases. Use the same SF Symbols as the current tabItem labels (calendar, checklist, bubble.left.and.bubble.right).
    - Set `.navigationTitle("Loom Cal")` on the sidebar list.
    - Detail pane: switch on selectedSection to show MacCalendarLayout / TaskListTabView / ChatView (wrapped in NavigationStack where needed for toolbar support).
    - Move the existing macOS toolbar items (plus menu for new event/task) into the detail-level toolbar for the calendar section.
    - Chat detail: include the existing toolbar (LoomAvatar + ellipsis menu).

    **iOS branch — existing TabView (unchanged):**
    - Keep the entire existing TabView code as-is, including the `#if !os(macOS)` FAB overlay.
    - Remove the inner `#if os(macOS)` / `#else` branch inside the Calendar tab — on iOS only the iOS calendar content shows (the macOS MacCalendarLayout branch was inside this tab, but now macOS has its own top-level path).

    3. The `.task { }` modifier wiring ViewModels and starting subscriptions MUST remain applied to the outermost container on both platforms — it currently lives on the TabView; apply it to both the NavigationSplitView and TabView branches.

    4. Sheet modifiers (.sheet for event create, task create, event detail, task detail) MUST remain on the outermost container on both platforms so they work regardless of selected section.

    **Avoid:**
    - Do NOT create separate files — keep everything in ContentView.swift
    - Do NOT change MacCalendarLayout.swift — it stays as-is, just used inside the detail pane
    - Do NOT remove the `#if os(macOS)` guard from MacCalendarLayout.swift itself
    - Do NOT change any ViewModel logic
