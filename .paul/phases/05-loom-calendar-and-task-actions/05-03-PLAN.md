## AC-1: Confirmation Card Rendering
```gherkin
Given a pending_action message exists in chat_messages
When the ChatView renders the message list
Then the message renders as an ActionConfirmationCard (not a ChatBubbleView)
And the card shows action type icon, summary, and type-specific detail fields
```

## AC-2: Confirm Executes Mutation
```gherkin
Given a pending_action confirmation card is displayed
When the user taps "Confirm"
Then the card status changes to "Confirmed"
And the corresponding event/task is created/updated/deleted in Convex
And the change appears in the calendar or task list within 2 seconds
```

## AC-3: Cancel Dismisses Without Mutation
```gherkin
Given a pending_action confirmation card is displayed
When the user taps "Cancel"
Then the card status changes to "Cancelled"
And no mutation is executed in Convex
```

## AC-4: Undo Banner and Reversal
```gherkin
Given the user has just confirmed a create or update action
When the confirmation succeeds
Then an UndoBanner appears above the input bar with a 5-second countdown
And tapping "Undo" within the window reverses the mutation
And the card status changes to "Undone"
```

## AC-5: Highlight Pulse Animation
```gherkin
Given a mutation has been confirmed and executed
When the affected event or task appears in the calendar/task list
Then it displays a pulsing accent-color border animation (3 pulses over 1.5s)
```

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Verify end-to-end Loom action flow</name>
  <what-built>
Complete Phase 5 Loom Calendar & Task Actions system:
- Bridge sends calendar/task context to OpenClaw alongside each message
- Loom proposes actions via structured ACTION JSON in replies
- Pending action messages render as confirmation cards in the chat stream
- Confirm executes the mutation (event/task CRUD) in real-time
- Cancel dismisses the card without executing
- Undo banner appears for 5 seconds after confirm (for creates and updates)
- Highlight pulse animation on the affected calendar event or task row
  </what-built>
  <how-to-verify>
**Prerequisites:**
1. Convex dev server running (`npx convex dev`)
2. Bridge script running (`node bridge/loom-bridge.mjs`)
3. App running on iOS Simulator

**Test 1 — Create Event:**
1. Open Chat tab
2. Type: "Add a dentist appointment this Thursday at 3pm for 1 hour"
3. Verify: Confirmation card with calendar.badge.plus icon, event details (title, date, time, duration)
4. Tap "Confirm"
5. Verify: Card shows green checkmark + "Confirmed"
6. Verify: Undo banner appears at bottom with countdown
7. Switch to Calendar tab — verify event appears on Thursday's timeline
8. Verify: New event has highlight pulse animation (3 pulses)

**Test 2 — Create Task:**
1. Open Chat tab
2. Type: "Add a task to buy groceries, high priority, due tomorrow"
3. Verify: Confirmation card with checkmark.circle icon, task details
4. Tap "Confirm"
5. Switch to Tasks — verify task appears with correct priority and due date

**Test 3 — Cancel Action:**
1. Type: "Create an event called Test at noon tomorrow"
2. Verify: Confirmation card appears
3. Tap "Cancel"
4. Verify: Card shows gray X + "Cancelled"
5. Switch to Calendar — verify no "Test" event was created

**Test 4 — Edit Event:**
1. Type: "Move my dentist appointment to 4pm"
2. Verify: Card shows before/after diff (3pm strikethrough, 4pm in primary)
3. Tap "Confirm"
4. Verify: Calendar shows updated time

**Test 5 — Delete Event:**
1. Type: "Delete my dentist appointment"
2. Verify: Card shows event to be deleted with trash icon
3. Tap "Confirm"
4. Verify: Event disappears from calendar

**Test 6 — Undo:**
1. Type: "Create an event called Undo Test tomorrow at 2pm"
2. Tap "Confirm"
3. Within 5 seconds, tap "Undo" on the banner
4. Verify: Event disappears from calendar, card shows orange "Undone" state

**Note:** If Loom's system prompt needs tuning for reliable ACTION block output, note the issue — that's a prompt refinement, not a code bug.
  </how-to-verify>
