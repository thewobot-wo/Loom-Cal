<objective>
## Goal
Build the recurrence data model and client-side expansion engine — schema support for exception dates, RRULE parsing/generation for common patterns (daily, weekly, monthly), and automatic occurrence expansion in CalendarViewModel.

## Purpose
Recurring events are a core calendar feature. This plan lays the data foundation so that a single master event with an RRULE string automatically expands into virtual occurrences across calendar views. Without this, users must manually create each instance of a repeating event.

## Output
- Updated Convex schema with `exceptionDates` field on events
- `RecurrenceRule.swift` — RRULE parsing, generation, and date expansion for daily/weekly/monthly
- CalendarViewModel expansion logic — `events(for:)` returns virtual occurrences for recurring masters
- LoomEvent convenience init for creating virtual occurrence instances
</objective>

<task type="auto">
  <name>Task 2: Create RecurrenceRule model with RRULE parsing and date expansion</name>
  <files>LoomCal/Models/RecurrenceRule.swift</files>
  <action>
    Create new file `LoomCal/Models/RecurrenceRule.swift` containing:

    1. **Weekday enum** — `monday` through `sunday`, with RRULE abbreviation mapping (MO, TU, WE, TH, FR, SA, SU) and Calendar.current weekday number mapping.

    2. **RecurrenceFrequency enum** — `.daily`, `.weekly`, `.monthly` (no yearly for now).

    3. **RecurrenceRule struct** with properties:
       - `frequency: RecurrenceFrequency`
       - `interval: Int` (default 1 — every Nth day/week/month)
       - `byDay: Set<Weekday>` (relevant for weekly; which days of week)
       - `byMonthDay: Int?` (relevant for monthly; which day of month, e.g., 15)
       - `until: Date?` (end date, nil = repeats forever within expansion horizon)
       - `count: Int?` (max number of occurrences, nil = unlimited)

    4. **RRULE parsing** — `static func from(rrule: String) -> RecurrenceRule?`
       - Split on ";" to get key=value pairs
       - Parse FREQ → frequency
       - Parse INTERVAL → interval (default 1 if absent)
       - Parse BYDAY → byDay set (comma-separated weekday abbreviations)
       - Parse BYMONTHDAY → byMonthDay
       - Parse UNTIL → until date (ISO 8601 basic format: YYYYMMDD or YYYYMMDDTHHmmssZ)
       - Parse COUNT → count
       - Return nil if FREQ is missing or unrecognized

    5. **RRULE generation** — `func toRRULE() -> String`
       - Build "FREQ=DAILY/WEEKLY/MONTHLY" base
       - Append ";INTERVAL=N" if interval > 1
       - Append ";BYDAY=MO,WE,FR" if byDay is non-empty (sorted for determinism)
       - Append ";BYMONTHDAY=N" if byMonthDay is set
       - Append ";UNTIL=YYYYMMDDTHHmmssZ" if until is set
       - Append ";COUNT=N" if count is set

    6. **Date expansion** — `func occurrences(from startDate: Date, through endDate: Date, excluding exceptionDates: [Date] = []) -> [Date]`
       - Generate occurrence dates starting from startDate up through endDate
       - For .daily: advance by `interval` days each step
       - For .weekly: for each week (advancing by `interval` weeks), include dates for each day in byDay. If byDay is empty, use the weekday of startDate.
       - For .monthly: advance by `interval` months, using byMonthDay if set (else use startDate's day). Handle month-end edge cases (e.g., 31st in a 30-day month → skip or clamp to last day).
       - Stop if `count` occurrences reached or past `until` date
       - Filter out any date whose startOfDay matches an exceptionDate's startOfDay
       - Cap at 365 occurrences max as safety limit
       - Use Calendar.current for all date arithmetic

    7. **Convenience factory methods:**
       - `static func daily(interval: Int = 1) -> RecurrenceRule`
       - `static func weekly(days: Set<Weekday>, interval: Int = 1) -> RecurrenceRule`
       - `static func weekdays() -> RecurrenceRule` (Mon-Fri shortcut)
       - `static func monthly(dayOfMonth: Int, interval: Int = 1) -> RecurrenceRule`

    Avoid: Supporting YEARLY, BYSETPOS, BYMONTH combinations, EXRULE, or WKST — those are exotic and out of scope.
    Avoid: Using any third-party library for RRULE parsing — keep it self-contained.
  </action>
  <verify>File compiles in Xcode. RecurrenceRule.from(rrule: "FREQ=WEEKLY;BYDAY=MO,WE,FR") returns non-nil with correct properties. RecurrenceRule.daily().toRRULE() returns "FREQ=DAILY".</verify>
  <done>AC-1 satisfied: RRULE parsing works for daily/weekly/monthly patterns. AC-2 satisfied: toRRULE() generates valid strings that round-trip through from(rrule:).</done>
