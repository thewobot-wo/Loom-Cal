<objective>
## Goal
Add recurrence UI throughout the event lifecycle — a recurrence picker in event creation, recurrence display in event detail, and edit-this/edit-all + delete-this/delete-all handling for recurring event occurrences.

## Purpose
Plan 09-01 built the expansion engine (RecurrenceRule + CalendarViewModel expansion). This plan surfaces recurrence to the user: they can create recurring events, see recurrence info, and manage individual vs. series operations.

## Output
- EventCreationView: recurrence picker (None/Daily/Weekdays/Weekly/Monthly)
- EventDetailView: recurrence badge, delete this/all occurrence choice
- EventEditView: edit this/all occurrence handling
- CalendarViewModel: createEvent with rrule, occurrence-level mutation helpers
</objective>

## AC-1: Recurrence Picker in Creation
```gherkin
Given the user opens EventCreationView and expands details
When they tap the Repeat row
Then they see options: Never, Daily, Weekdays, Weekly, Monthly
And selecting one updates the recurrence state
And saving creates the event with the corresponding RRULE string
```

## AC-2: Recurrence Badge in Detail
```gherkin
Given a recurring event (rrule is non-nil)
When the user opens EventDetailView for any occurrence
Then a recurrence description is shown (e.g., "Repeats daily", "Repeats weekly on Mon, Wed, Fri")
```

## AC-3: Delete This Occurrence
```gherkin
Given a recurring event occurrence displayed in EventDetailView
When the user taps Delete and chooses "This Event"
Then an exception date is added to the master event for that occurrence's date
And the occurrence disappears from the calendar
And all other occurrences remain unchanged
```

## AC-4: Delete All Events
```gherkin
Given a recurring event occurrence displayed in EventDetailView
When the user taps Delete and chooses "All Events"
Then the master event is deleted from Convex
And all occurrences disappear from the calendar
```

## AC-5: Edit This / Edit All Choice
```gherkin
Given a recurring event occurrence displayed in EventDetailView
When the user taps Edit
Then they are presented with "This Event" / "All Events" choice
And choosing "This Event" creates a standalone copy at this occurrence's date + adds exception to master
And choosing "All Events" edits the master event directly
```
