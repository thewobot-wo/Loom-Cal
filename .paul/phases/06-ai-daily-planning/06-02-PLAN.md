<acceptance_criteria>

## AC-1: Daily Plan Card Renders Time Blocks
```gherkin
Given a pending_action message with type "daily_plan" containing 3 proposed blocks
When the ChatView renders this message
Then a DailyPlanCard is shown (not an ActionConfirmationCard)
And the card displays the plan summary text
And each block shows its title, formatted start time, and duration
And blocks are listed in chronological order
```

## AC-2: Pending Plan Shows Approve/Reject Buttons
```gherkin
Given a DailyPlanCard for a message with actionStatus "pending"
When the card is displayed
Then "Approve" and "Reject" buttons are visible
And tapping "Approve" calls chatViewModel.approveDailyPlan(message)
And tapping "Reject" calls chatViewModel.rejectDailyPlan(message)
```

## AC-3: Resolved Plan Collapses to Status Line
```gherkin
Given a DailyPlanCard for a message with actionStatus "confirmed", "cancelled", or "undone"
When the card is rendered
Then the card collapses to a compact single line
And the line shows a status icon (checkmark/x/undo arrow) plus the plan summary
And no Approve/Reject buttons are shown
```

## AC-4: ChatView Routes Daily Plans Correctly
```gherkin
Given the chat message list contains both regular actions and daily plans
When ChatView renders pending_action messages
Then messages where isDailyPlan is true render as DailyPlanCard
And messages where isDailyPlan is false render as ActionConfirmationCard
And both card types coexist in the same chat stream
```

  <name>Task 1: Create DailyPlanCard component</name>
  <files>LoomCal/Views/Chat/DailyPlanCard.swift, LoomCal.xcodeproj/project.pbxproj</files>
  <action>
    Create `LoomCal/Views/Chat/DailyPlanCard.swift` following the ActionConfirmationCard pattern:

    **Structure:**
    - Takes `message: ChatMessage`, `onApprove: () -> Void`, `onReject: () -> Void`
    - Private computed `plan: DailyPlanProposal?` from `message.decodedPlan`
    - Private computed `actionStatus: String` from `message.actionStatus ?? "pending"`
    - Body switches on actionStatus: pending → expandedCard, else → collapsedCard

    **Expanded card (pending state):**
    - Left-aligned VStack inside an HStack (match ActionConfirmationCard layout)
    - Header: calendar.day.timeline.leading icon + "DAILY PLAN" label in accent color
    - Summary text: `plan.displaySummary` in .body font
    - Blocks list: ForEach over `plan.payload.blocks` showing each PlannedBlock:
      - Each row: HStack with clock icon, formatted time range, and title
      - Time format: "9:00 AM - 10:00 AM" using block.startDate and block.endDate
      - Duration badge: block.durationFormatted in .caption2 with pill background
    - Approve/Reject buttons at bottom (same style as ActionConfirmationCard):
      - "Approve" → .borderedProminent with checkmark icon
      - "Reject" → .bordered with .secondary tint
      - Include UIImpactFeedbackGenerator behind `#if canImport(UIKit)`
    - Card background: same RoundedRectangle fill/stroke as ActionConfirmationCard
    - Padding: .leading 8, .trailing 60 (match ActionConfirmationCard)

    **Collapsed card (confirmed/cancelled/undone):**
    - Same pattern as ActionConfirmationCard.collapsedCard
    - Status icon (checkmark.circle.fill green / xmark.circle.fill secondary / arrow.uturn.backward.circle.fill orange)
    - Summary text in .caption, single line

    **Time formatter:**
    - Private static DateFormatter with timeStyle .short (gives "9:00 AM")
    - Use block.startDate and block.endDate for the time range

    Add DailyPlanCard.swift to the Xcode project (pbxproj) in the same group as ActionConfirmationCard.swift.

    Avoid:
    - Do NOT duplicate shared formatters — the card-specific formatters are fine since they differ from ActionConfirmationCard's
    - Do NOT add scroll behavior inside the card — it lives in ChatView's ScrollView
    - Do NOT add animation beyond the existing chat scroll behavior
