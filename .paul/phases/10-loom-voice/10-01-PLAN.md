<objective>
## Goal
Add Loom voice output: bridge generates TTS via OpenClaw, uploads audio to Convex file storage, and the iOS/macOS app plays it with per-message play/pause controls and a global voice toggle.

## Purpose
Gives Loom an audible voice — users can listen to responses hands-free. Audio is generated server-side (bridge) using the same OpenClaw voice service already working in Telegram.

## Output
- Modified `bridge/loom-bridge.mjs` — TTS generation + Convex file upload + extended reply payload
- Modified `convex/schema.ts` — audio storage field on chat_messages

## AC-1: Bridge TTS Generation
```gherkin
Given the bridge receives a text reply from OpenClaw
When ENABLE_TTS is true (default)
Then the bridge calls OpenClaw /v1/audio/speech with the reply text
And uploads the audio to Convex file storage
And posts the reply with both content and audioStorageId to /loom-reply
```

## AC-2: Audio Playback in Chat
```gherkin
Given an assistant message has an audioUrl (resolved from Convex storage)
When the user taps the play button on that message
Then the audio plays through the device speaker
And the button shows a pause icon while playing
And the button returns to play icon when audio finishes
```

## AC-3: Playback Controls
```gherkin
Given audio is playing for a message
When the user taps pause
Then audio pauses at current position
When the user taps play again
Then audio resumes from where it paused
When the user taps play on a DIFFERENT message
Then the previous audio stops and the new message's audio begins
```

## AC-4: Voice Toggle Preference
```gherkin
Given the user enables the voice toggle in the chat toolbar
When a new assistant message arrives with audio
Then the audio auto-plays
Given the user disables the voice toggle
When a new assistant message arrives
Then no auto-play occurs (manual play buttons still work)
And the toggle state persists across app launches via UserDefaults
```

## AC-5: Cross-Platform Audio
```gherkin
Given the app is running on iOS
When audio plays from a URL
Then AVAudioSession is configured for playback (mixWithOthers)
Given the app is running on macOS
When audio plays from a URL
Then audio plays through default output (no AVAudioSession needed)
```

  <name>Task 1: Bridge TTS + Convex backend audio support</name>
  <files>bridge/loom-bridge.mjs, convex/schema.ts, convex/chatMessages.ts, convex/http.ts</files>
  <action>
    **convex/schema.ts** — add audio field to chat_messages:
    - Add `audioStorageId: v.optional(v.id("_storage"))` to chat_messages table

    **convex/chatMessages.ts** — update mutations and queries:
    - `writeAssistantReply`: add optional `audioStorageId: v.optional(v.id("_storage"))` arg, store it on insert
    - `list` query: resolve audioUrl for each message that has audioStorageId using `ctx.storage.getUrl(msg.audioStorageId)`. Return audioUrl as a string field alongside existing fields.

    **convex/http.ts** — update /loom-reply + add upload endpoint:
    - `/loom-reply` handler: accept optional `audioStorageId` string in body, pass to writeAssistantReply
    - New `POST /generate-upload-url` endpoint: calls `ctx.storage.generateUploadUrl()`, returns `{ url }`. Same auth pattern as other endpoints.

    **bridge/loom-bridge.mjs** — add TTS step after text reply:
    - New env var: `ENABLE_TTS` (default: "true"). When "false", skip TTS entirely.
    - New env var: `TTS_VOICE` (default: "shimmer"). Voice name for OpenClaw TTS.
    - New env var: `TTS_MODEL` (default: "tts-1"). Model for OpenClaw TTS.
    - New async function `generateAndUploadTTS(text)`:
      1. Strip markdown from text (remove **, *, #, `, link syntax []())
      2. Call `${OPENCLAW_URL}/v1/audio/speech` with POST:
         - Headers: `Authorization: Bearer ${currentPassword}`, `Content-Type: application/json`
         - Body: `{ "model": TTS_MODEL, "voice": TTS_VOICE, "input": strippedText }`
         - Response: raw audio bytes
      3. Get upload URL: `GET ${CONVEX_SITE_URL}/generate-upload-url`
      4. Upload audio: `POST` to the upload URL with audio data, `Content-Type: audio/mpeg`
         - Convex upload returns `{ storageId: "..." }`
      5. Return storageId (or null on any failure — TTS is non-blocking, text reply still posts)
    - In `poll()` after getting replyText and before posting:
      - If ENABLE_TTS and no action block detected:
        `const audioStorageId = await generateAndUploadTTS(replyText)`
    - Update `postLoomReply(content)` → `postLoomReply(content, audioStorageId)`:
      Send `{ content, audioStorageId }` (audioStorageId optional/null OK)
    - Log TTS success/failure for debugging
    - TTS errors must NOT block the text reply — catch and continue

    Avoid: Making TTS a blocking requirement. If TTS fails, the text reply still gets posted.
    Avoid: Generating TTS for ACTION block messages — only for regular text replies.
  </action>

    Bridge starts without errors with ENABLE_TTS=true.
    Test: send a message to Loom, check bridge logs for TTS generation + upload.

<task type="auto">
  <name>Task 2: iOS audio playback + voice UI</name>
  <files>LoomCal/Models/ChatMessage.swift, LoomCal/Services/VoiceService.swift (new), LoomCal/Views/Chat/ChatBubbleView.swift, LoomCal/Views/Chat/ChatView.swift, LoomCal/ViewModels/ChatViewModel.swift</files>
  <action>
    **ChatMessage.swift** — add audio URL field:
    - Add `let audioUrl: String?` (nullable — only present when TTS was generated)
    - This is resolved server-side by the list query, not a raw storageId

    **VoiceService.swift** — new file in Services/, URL-based audio player:
    - `@MainActor class VoiceService: NSObject, ObservableObject, AVAudioPlayerDelegate`
    - `@Published var playingMessageId: String?` (nil = idle)
    - `@Published var isLoadingAudio: Bool = false` (true while downloading)
    - Private `audioPlayer: AVAudioPlayer?`
    - Private `audioCache: [String: Data]` — in-memory cache keyed by message ID

    Core methods:
    - `func play(url: String, messageId: String)`:
      1. If same messageId is paused → resume (`audioPlayer?.play()`)
      2. If different messageId is playing → stop current, start new
      3. Check `audioCache[messageId]` — if cached, play from cache
      4. Otherwise: set isLoadingAudio = true, download from URL via URLSession, cache, play
      5. On iOS: configure AVAudioSession `.playback` category with `.mixWithOthers`
         Use `#if canImport(UIKit)` guard
    - `func pause()`: pause audioPlayer, keep playingMessageId
    - `func stop()`: stop audioPlayer, set playingMessageId = nil
    - `func togglePlayPause(url: String, messageId: String)`:
      If currently playing this messageId → pause. If paused → resume. Otherwise → play.
    - AVAudioPlayerDelegate: `audioPlayerDidFinishPlaying` → set playingMessageId = nil

    Import AVFoundation. No ElevenLabs. No API keys on device.

    **ChatViewModel.swift** — add voice coordination:
    - Add: `let voiceService = VoiceService()`
    - Add: `@AppStorage("loomVoiceEnabled") var voiceEnabled: Bool = false`
    - Add method `playMessage(_ message: ChatMessage)`:
      Guard `message.audioUrl != nil`, then call `voiceService.togglePlayPause(url:messageId:)`
    - In `startSubscription()`, after detecting a new assistant message:
      If `voiceEnabled` and new message has audioUrl → call `voiceService.play(url:messageId:)`

    **ChatBubbleView.swift** — add play button for assistant messages with audio:
    - Add parameters: `voiceService: VoiceService`, `onPlayTap: (() -> Void)?`
    - Below the assistant bubble (outside the background), if message has audio:
      ```
      HStack(spacing: 4) {
          Button { onPlayTap?() } label: {
              if voiceService.isLoadingAudio && voiceService.playingMessageId == message._id {
                  ProgressView().controlSize(.small)
              } else {
                  Image(systemName: isThisPlaying ? "pause.circle.fill" : "play.circle.fill")
              }
          }
          .font(.caption)
          .foregroundStyle(.secondary)
      }
      ```
    - `isThisPlaying` = `voiceService.playingMessageId == message._id`
    - Only show play button when `message.audioUrl != nil` AND `!isUser`

    **ChatView.swift** — wire up voice service and toggle:
    - Pass `voiceService` and `onPlayTap` to ChatBubbleView
    - Add voice toggle button in toolbar area:
      `Image(systemName: voiceEnabled ? "speaker.wave.2.fill" : "speaker.slash.fill")`
      Binds to `chatViewModel.voiceEnabled`

    Avoid: Adding play buttons to user messages or action cards.
    Avoid: Auto-playing TTS for pending_action messages.
  </action>
