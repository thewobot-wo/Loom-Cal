Run `npm install convex` to add the Convex dependency.

Create `convex/schema.ts` with all four tables defined per the locked decisions in CONTEXT.md:

**events table:**
- calendarId: v.string() — links to a named calendar/source
- title: v.string()
- start: v.int64() — UTC milliseconds
- duration: v.int64() — minutes
- timezone: v.string() — IANA timezone (e.g. "America/New_York")
- isAllDay: v.boolean()
- location: v.optional(v.string())
- notes: v.optional(v.string()) — markdown plain text
- url: v.optional(v.string()) — dedicated meeting link field
- color: v.optional(v.string()) — user-picked color from palette
- rrule: v.optional(v.string()) — RRULE string for recurrence (schema field included per user decision; UI deferred)
- recurrenceGroupId: v.optional(v.string()) — links recurring instances
- attachments: v.optional(v.array(v.string())) — file references (schema included; upload UI deferred)
- Indexes: by_calendar on ["calendarId"], by_start on ["start"]

**tasks table:**
- title: v.string()
- dueDate: v.optional(v.int64()) — UTC milliseconds
- flagged: v.boolean() — boolean flag only, NOT priority tiers (per user decision)
- completed: v.boolean()
- notes: v.optional(v.string()) — markdown plain text
- attachments: v.optional(v.array(v.string()))
- Indexes: by_due_date on ["dueDate"], by_completed on ["completed"]

**chat_messages table:**
- role: v.union(v.literal("user"), v.literal("assistant"))
- content: v.string()
- sentAt: v.int64() — UTC milliseconds
- Index: by_sent_at on ["sentAt"]

**studio_events table:**
- calendarId: v.string() — fixed value like "studio"
- title: v.string()
- start: v.int64() — UTC milliseconds
- duration: v.int64() — minutes
- timezone: v.string()
- isAllDay: v.boolean()
- lastSyncedAt: v.int64() — UTC ms timestamp of last Supabase sync
- Index: by_start on ["start"]

Add a comment at the top of schema.ts: `// v.int64() fields require BigInt() in mutations and @ConvexInt in Swift structs`

CRITICAL: Use v.int64() for ALL integer fields (timestamps, durations). Do NOT use v.number() — it is float64 and causes Swift deserialization issues.

<task type="auto">
  <name>Task 2: Create query and mutation functions for all tables</name>
  <files>convex/events.ts, convex/tasks.ts, convex/chatMessages.ts, convex/studioEvents.ts, convex/crons.ts</files>
  <action>
Create Convex function files for each table. Use `query`, `mutation`, `internalAction`, and `internalMutation` from `convex/server`. Reference the research patterns from 01-RESEARCH.md.

**convex/events.ts:**
- `list` query: returns all events ordered by start time using `withIndex("by_start")`
- `create` mutation: accepts all event fields, inserts into events table. All v.int64() args must be typed correctly in the args validator.
- `update` mutation: accepts event _id + partial fields, patches the event
- `remove` mutation: accepts event _id, deletes it

**convex/tasks.ts:**
- `list` query: returns all tasks ordered by due date using `withIndex("by_due_date")`
- `create` mutation: accepts all task fields, inserts into tasks table
- `update` mutation: accepts task _id + partial fields, patches the task
- `remove` mutation: accepts task _id, deletes it

**convex/chatMessages.ts:**
- `list` query: returns all messages ordered by sentAt using `withIndex("by_sent_at")`
- `send` mutation: accepts role and content, inserts with sentAt = BigInt(Date.now())

**convex/studioEvents.ts:**
- `list` query: returns all studio events ordered by start time using `withIndex("by_start")`
- `syncFromSupabase` internalAction: fetches from `process.env.SUPABASE_EVENTS_URL` with `apikey` header from `process.env.SUPABASE_ANON_KEY`, calls `ctx.runMutation(internal.studioEvents.upsertAll, { rows })`. Include error handling — if fetch fails, log error and return without crashing the cron.
- `upsertAll` internalMutation: accepts array of rows, upserts into studio_events table. Use a sourceId field or title match for deduplication. All v.int64() field values must use BigInt() explicitly.

**convex/crons.ts:**
- Register a cron job with `cronJobs()` that runs `internal.studioEvents.syncFromSupabase` every 15 minutes.

IMPORTANT: In all mutations that write v.int64() fields, pass values as `BigInt(value)`. Do NOT pass plain numbers to v.int64() fields — Convex will throw ArgumentValidationError.

For the `update` mutations, use `v.optional()` for each updatable field in the args validator so callers only need to pass changed fields. Use `ctx.db.patch(id, updates)` to apply partial updates.
  </action>
