**TaskRowView.swift — Things 3-style compact row (per CONTEXT.md locked decision):**

Structure: `HStack(spacing: 0)` with:
1. Priority left edge bar — `Rectangle().fill(task.priorityColor).frame(width: 3)` (red=high, yellow=medium, blue/low)
2. Completion circle button — `Image(systemName: task.completed ? "checkmark.circle.fill" : "circle")` colored by priority; `.buttonStyle(.plain)` to avoid row tap interference
3. Title text — `.strikethrough(task.completed)`, `.foregroundStyle(task.completed ? .secondary : .primary)`
4. Due date chip — `Text(task.dueDateFormatted)` in `.font(.caption2)`, red if overdue, secondary otherwise
5. `Spacer()` to push content left

Full row has `.background(.background)` and `.contentShape(Rectangle())` for tap area.

Callbacks: `var onComplete: () -> Void` and `var onTap: () -> Void`.

Completion animation: When `onComplete` fires, the parent view should call `withAnimation(.easeInOut(duration: 0.25))` to toggle completion state, triggering the strikethrough+fade. The animation is driven by the `completed` property changing on the model (from Convex subscription update).

**TaskCreationView.swift — creation form (Claude's Discretion for form design):**

Structure: `NavigationStack` with `Form`:
- Section 1: Title `TextField("Task name", text: $title)` — required, disable save if empty
- Section 2: Priority `Picker("Priority", selection: $priority)` with segmented style, options: "High", "Medium", "Low" (default: "low")
- Section 3: Due Date toggle `Toggle("Due date", isOn: $hasDueDate)`, if on show `DatePicker` for date. Sub-toggle: `Toggle("Specific time", isOn: $hasDueTime)`, if on show time `DatePicker`
- Section 4 (optional): Notes `TextField("Notes", text: $notes, axis: .vertical).lineLimit(3...6)`

Toolbar: Cancel (dismiss) and Save button. Save calls `taskViewModel.createTask(title:priority:dueDate:hasDueTime:notes:)` then dismisses.

When hasDueDate is false, dueDate is nil. When hasDueDate is true but hasDueTime is false, pass midnight of the selected day. When both are true, pass the full date+time.

Props: `@ObservedObject var taskViewModel: TaskViewModel`, `@Binding var isPresented: Bool`, optional `prefilledDate: Date?` (if provided, auto-enable hasDueDate and set date).

**pbxproj:** Create Tasks group under Views. Add both Swift files to the project following the same pattern as the Events group (PBXFileReference, PBXBuildFile, PBXGroup, PBXSourcesBuildPhase).

**TaskDetailView.swift — combined detail + edit view:**

Structure: `NavigationStack` sheet with `Form`:
- Section "Details":

  - Priority: `Picker` segmented with "High"/"Medium"/"Low"

- Section "Actions":
  - Complete/Uncomplete button: `Button(task.completed ? "Mark Incomplete" : "Mark Complete")` — calls `taskViewModel.toggleComplete(task:)` then dismisses

Save logic (mirrors EventEditView pattern): Compare each field against original task. Only pass changed fields to `taskViewModel.updateTask(id:title:priority:dueDate:hasDueTime:notes:)`. Dismiss after successful save.

The completion animation with undo is handled at the list level (in TodayView/Plan 03), not here. TaskDetailView just toggles the completed state and dismisses.

Init: Pre-fill all `@State` fields from the `task` parameter in `init()` using `@State(initialValue:)` pattern (same as EventEditView).
