**TodayView.swift — unified event+task timeline (per CONTEXT.md locked decisions):**

TodayView is structurally based on DayTimelineView but adds task integration. It receives both `calendarViewModel` and `taskViewModel`.

Layout structure (follows GeometryReader-as-root pattern from DayTimelineView):
```
GeometryReader { geo in
    VStack(spacing: 0) {
        // 1. Unscheduled tasks section (fixed height, above ScrollView)
        if !unscheduledTasks.isEmpty {
            UnscheduledTaskSection (collapsible)
        }
        Divider() (only if unscheduled tasks exist)

        // 2. ScrollView with timeline
        ScrollView(.vertical) {
            ZStack(alignment: .topLeading) {
                Color.clear.frame(height: totalHeight)  // height spacer
                // Hour grid lines + labels
                // Now indicator
                // Event cards (same as DayTimelineView)
                // Task time markers (compact rows at due time)
            }
        }
    }
}
```

**Unscheduled Tasks Section:**
- Shows tasks due today (or overdue) that have NO specific time (`!hasDueTime`)
- Capped at 3 visible rows. If more, show "Show all (N)" toggle
- Each row is a `TaskRowView` with `onComplete` calling `taskViewModel.toggleComplete(task:)` and `onTap` opening the task detail sheet
- Header: "Tasks" label + count badge
- Collapsible via `@State private var isTaskSectionExpanded = true`

**Timeline integration:**
- Events: render exactly like DayTimelineView (TimelineEventCard)
- Tasks with due time: render as compact inline rows at their time slot. Use a lighter visual treatment than events:
  - `HStack`: priority dot (5pt circle, priorityColor) + title text (.font(.caption), .foregroundStyle(.secondary)) + completion circle
  - Position using same `yOffset(for:)` helper as events — convert dueDate ms to Date, extract hour/minute
  - Height: fixed 20pt (compact)
  - Background: Color.clear (no card background — just inline text)

**Interleaving sort logic:**
Create a `TimelineItem` enum:
```swift
enum TimelineItem: Identifiable {
    case event(LoomEvent)
    case task(LoomTask)

    var id: String { ... }
    var startDate: Date { ... }
}
```
Merge events and timed tasks, sort by startDate. Render each item in the ZStack at its computed Y offset.

**Event tap and task tap:** Both open sheets — event detail sheet and task detail sheet respectively. Use `@State private var selectedEvent: LoomEvent?` and `@State private var selectedTask: LoomTask?` with separate `.sheet(item:)` modifiers.

**Task creation:** Include a floating "+" button or toolbar button that opens TaskCreationView sheet. Also support event creation via the existing `showCreateSheet` pattern.

**Props:** `@ObservedObject var calendarViewModel: CalendarViewModel`, `@ObservedObject var taskViewModel: TaskViewModel`, `onEventTap: (LoomEvent) -> Void`, `onEventDragMove` (same as DayTimelineView for event drag).
