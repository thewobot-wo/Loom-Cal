1. **WeekTimelineView.swift** (in Views/Calendar/):
   - Accept `@ObservedObject var viewModel: CalendarViewModel` and `onEventTap: (LoomEvent) -> Void`.
   - Compute the 7 dates for the week containing `viewModel.selectedDate`: use `Calendar.current.dateInterval(of: .weekOfYear, for: viewModel.selectedDate)` to get week start, then iterate 0..<7 adding days.
   - Layout: `ScrollView(.vertical)` containing a `HStack(spacing: 0)` of 7 day columns.
   - Each column:
     - Day header at top: abbreviated weekday name (.caption, "Mon", "Tue") + day number. Highlight today's column with accent color. Highlight selected date.
     - Below header: a mini vertical timeline (same concept as DayTimelineView but compressed).
     - `let pointsPerHour: CGFloat = 50.0` (slightly smaller than day view to fit 7 columns).
     - Event cards in week view: for narrow columns (<60pt), show a colored bar only (no text) per RESEARCH open question #3. For wider columns (iPad/Mac), show abbreviated title.
     - Width: divide available width equally among 7 columns using `GeometryReader`. Subtract space for a narrow time label gutter on the leading side (shared across all columns, ~35pt).
   - Time labels: shared leading gutter with hour markers (every 2 hours to reduce clutter: 12a, 2a, 4a... or just major hours). Format 12-hour.
   - NowIndicatorView: horizontal red line spanning all 7 columns at current time offset.
   - `onEventTap` callback passed through to event card taps.
   - All-day events: small colored dots or abbreviated banner at column top (keep minimal for space).

2. **ContentView.swift** — FULL REPLACEMENT of the Phase 1 proof-of-concept:
   - `@StateObject private var viewModel = CalendarViewModel()`
   - `@State private var viewMode: ViewMode = .day` — enum with `.day` and `.week` cases.
   - `@State private var showCreateSheet = false`
   - `@State private var selectedEvent: LoomEvent? = nil` — triggers detail sheet
   - `@State private var createPrefilledDate: Date? = nil` — for long-press creation
   - Layout per locked decisions (Fantastical-style):
     ```
     NavigationStack {
       VStack(spacing: 0) {
         // Segmented control
         Picker("View", selection: $viewMode) {
           Text("Day").tag(ViewMode.day)
           Text("Week").tag(ViewMode.week)
         }
         .pickerStyle(.segmented)
         .padding(.horizontal)

         // Mini month (always visible)
         MiniMonthView(viewModel: viewModel, onDateLongPress: { date in
           createPrefilledDate = date
           showCreateSheet = true
         })

         // Timeline (day or week based on segmented control)
         switch viewMode {
         case .day:
           DayTimelineView(
             events: viewModel.timedEvents(for: viewModel.selectedDate),
             allDayEvents: viewModel.allDayEvents(for: viewModel.selectedDate),
             onEventTap: { event in selectedEvent = event }
           )
         case .week:
           WeekTimelineView(
             viewModel: viewModel,
             onEventTap: { event in selectedEvent = event }
           )
         }
       }
       .navigationTitle("Loom Cal")
       .navigationBarTitleDisplayMode(.inline)
       .toolbar {
         ToolbarItem(placement: .primaryAction) {
           Button(action: { showCreateSheet = true }) {
             Image(systemName: "plus")
           }
         }
         ToolbarItem(placement: .navigation) {
           Button("Today") {
             viewModel.selectedDate = .now
           }
         }
       }
     }
     ```
   - Swipe navigation: wrap the timeline area in a `TabView` with `.tabViewStyle(.page(indexDisplayMode: .never))` OR use a custom gesture approach. The simpler approach: attach `.gesture(DragGesture(...).onEnded { ... })` to the timeline container. On swipe left (negative x translation > threshold), advance selectedDate by 1 day (day mode) or 7 days (week mode). On swipe right (positive x translation > threshold), go back. Use `withAnimation` for smooth transition.
   - Drag-to-move on day timeline: The DayTimelineView from Plan 01 has event cards. Add `DragGesture` to TimelineEventCard within DayTimelineView. On drag, compute vertical offset in points -> minutes delta -> new start time. Snap to nearest 15-minute boundary. On `.onEnded`, call `viewModel.updateEvent(id:, start: newStart)`. Show a shadow/opacity change during drag to indicate movement. This requires DayTimelineView to accept the viewModel (or an update callback). If DayTimelineView needs modification, update it here.
   - Sheet presentations:
     ```swift
     .sheet(isPresented: $showCreateSheet) {
       EventCreationView(viewModel: viewModel, isPresented: $showCreateSheet, prefilledDate: createPrefilledDate)
     }
     .sheet(item: $selectedEvent) { event in
       EventDetailView(event: event, viewModel: viewModel, isPresented: .init(
         get: { selectedEvent != nil },
         set: { if !$0 { selectedEvent = nil } }
       ))
     }
     ```
     Note: For `.sheet(item:)` to work, LoomEvent must conform to `Identifiable`. It already has `_id: String` — add `var id: String { _id }` if not present, or use a wrapper.
   - `.task { viewModel.startSubscription() }` — start Convex subscription on view appear.
   - `.onChange(of: showCreateSheet) { if !showCreateSheet { createPrefilledDate = nil } }` — reset prefilled date when sheet closes.
