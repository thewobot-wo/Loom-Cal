   - Convert `event.start` (Int, ms) to Date: `Date(timeIntervalSince1970: TimeInterval(event.start) / 1000)`.
   - Visual style per locked decisions: rounded rectangle background (cornerRadius: 8), subtle shadow (.shadow(color: .black.opacity(0.08), radius: 4, y: 2)), padding(8). Background color: `.blue.opacity(0.15)` with `.blue` leading accent bar (4pt wide rounded rect on leading edge).
   - Clean, generous whitespace — Fantastical-inspired.
   - Add an `onTap: () -> Void` callback for event detail navigation (wired in Plan 02).

3. **AllDayBannerView.swift**:
   - Accepts `[LoomEvent]` (pre-filtered to all-day events).
   - Horizontal ScrollView of compact pill-shaped labels (title only, .caption font).
   - Each pill: rounded rectangle background, `.blue.opacity(0.12)`, padding(.horizontal, 8), padding(.vertical, 4).
   - If no all-day events, render `EmptyView()` (zero height).

4. **DayTimelineView.swift**:
   - Accepts `events: [LoomEvent]` (pre-filtered for the day, timed events only) and `allDayEvents: [LoomEvent]`.

   - `private var totalHeight: CGFloat { 24 * pointsPerHour }` = 1440 pt.
   - Layout structure:
     ```

       ScrollView {
         ZStack(alignment: .topLeading) {
           // Hour grid lines + labels
           // Now indicator
           // Event cards (absolute positioned)
         }
         .frame(height: totalHeight)
       }
     }
     ```
   - Hour grid: `ForEach(0..<24)` — each row has a time label on the leading side (formatted "h a" for round hours, 12-hour format) and a thin gray Divider line. Position each at `CGFloat(hour) * pointsPerHour`.
   - Hour labels: `.font(.caption2)`, `.foregroundStyle(.secondary)`, fixed width ~50pt on leading edge. Timeline content area starts after labels.
   - NowIndicatorView positioned at computed y-offset for current time, spanning the content area (not overlapping hour labels).
   - Event cards: For each event, compute y-offset from event start time and height from duration. Use `GeometryReader` or fixed width calculation for the content area width. Handle overlapping events: implement a simple column assignment algorithm — sort events by start, greedily assign to columns (if event overlaps previous in same column, move to next column). Divide available width by number of columns in overlap group. This is Claude's Discretion — side-by-side layout preferred.
   - Auto-scroll to current time on appear: Use `ScrollPosition` (iOS 18 API per RESEARCH). Initialize `@State private var scrollPosition = ScrollPosition(y: 0)`. On `.onAppear`, set `scrollPosition = ScrollPosition(y: currentTimeYOffset - 100)` (offset up a bit so current time is not at the very top).
   - Add `onEventTap: (LoomEvent) -> Void` callback.
