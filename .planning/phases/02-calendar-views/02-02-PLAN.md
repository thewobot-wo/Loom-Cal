1. **NLEventParser.swift** (in Services/):
   - `struct NLEventParser` with `static func parse(_ input: String) -> ParsedEvent`.
   - `struct ParsedEvent` with: `title: String`, `date: Date?`, `hasTime: Bool`.
   - Use `NSDataDetector(types: NSTextCheckingResult.CheckingType.date.rawValue)` to find date/time in the input string.
   - `enumerateMatches` to find the first `.date` result. Extract `match.date` as the detected date.
   - Title cleanup: get the `match.range` from the detector result and remove that range from the original string. Also strip common time patterns via regex: `\b\d{1,2}(:\d{2})?\s*(am|pm)\b`, `\b(today|tomorrow|tonight)\b` (case insensitive). Trim whitespace. If resulting title is empty, use original input.
   - Determine `hasTime`: if `NSDataDetector` found a date AND the detected date has a non-midnight time component, set `hasTime = true`. Otherwise `false`.
   - Test cases the parser should handle correctly:
     - "Dentist 3pm" -> title: "Dentist", date: today 3:00 PM, hasTime: true
     - "Team meeting tomorrow 10am" -> title: "Team meeting", date: tomorrow 10:00 AM, hasTime: true
     - "Birthday party" -> title: "Birthday party", date: nil, hasTime: false

2. **EventCreationView.swift** (in Views/Events/):
   - Presented as a `.sheet` (triggered by plus button or mini month long-press from parent).
   - Accept `@ObservedObject var viewModel: CalendarViewModel`, `@Binding var isPresented: Bool`, `var prefilledDate: Date?` (for long-press creation).
   - Layout per locked decisions:
     ```
     NavigationStack {
       Form {
         // NL input section
         Section {
           TextField("Dentist 3pm, Team lunch tomorrow...", text: $nlInput)
             .font(.title3)
             .onSubmit { parseAndFill() }
         }

         // Parsed/manual details section (expandable)
         Section("Details") {
           TextField("Title", text: $title)
           DatePicker("Date", selection: $eventDate, displayedComponents: .date)
           DatePicker("Start Time", selection: $startTime, displayedComponents: .hourAndMinute)
           Picker("Duration", selection: $durationMinutes) {
             // 15, 30, 45, 60, 90, 120 minutes
           }
           Toggle("All Day", isOn: $isAllDay)
         }
       }
       .navigationTitle("New Event")
       .navigationBarTitleDisplayMode(.inline)
       .toolbar {
         ToolbarItem(placement: .cancellationAction) { Button("Cancel") { isPresented = false } }
         ToolbarItem(placement: .confirmationAction) {
           Button("Add") { saveEvent() }
             .disabled(title.isEmpty)
         }
       }
     }
     ```
   - `@State private var nlInput = ""`, `title = ""`, `eventDate = Date()`, `startTime = Date()`, `durationMinutes = 60` (default 1 hour per locked decision), `isAllDay = false`.
   - If `prefilledDate` is provided, initialize `eventDate` to that date.
   - `parseAndFill()`: call `NLEventParser.parse(nlInput)`. If result has title, set `title`. If result has date with time, set both `eventDate` and `startTime`. If date only (no time), set `eventDate` only.
   - `saveEvent()`: combine `eventDate` (day) and `startTime` (hour/minute) into a single Date. Call `viewModel.createEvent(title:start:durationMinutes:isAllDay:)`. Dismiss sheet on success. Wrap in Task { try await ... } with basic error handling (print for now, error toast is Claude's Discretion).
   - Use `.datePickerStyle(.compact)` for both DatePickers — native and compact per RESEARCH anti-patterns.
   - When `isAllDay` toggled on, hide the Start Time and Duration pickers.
   - NL input field should call `parseAndFill()` as the user types (with debounce) OR on submit. On-submit is simpler and avoids excessive parsing. Use `.onSubmit`.

   - Accept `let event: LoomEvent`, `@ObservedObject var viewModel: CalendarViewModel`, `@Binding var isPresented: Bool`.
   - Layout:
     ```
     NavigationStack {
       VStack(alignment: .leading, spacing: 16) {
         // Event title (large, bold)
         Text(event.title).font(.title2).bold()

         // Date and time info
         HStack {
           Image(systemName: "calendar")
           Text(formattedDate)  // "Thursday, February 20, 2026"
         }
         HStack {
           Image(systemName: "clock")
           Text(formattedTimeRange)  // "3:00 PM - 4:00 PM" (start + duration)

           Label("All Day", systemImage: "sun.max")
         }

         Spacer()

         // Action buttons
         HStack {
           Button("Edit") { showEditSheet = true }
             .buttonStyle(.bordered)
           Spacer()
           Button("Delete", role: .destructive) { showDeleteConfirm = true }
             .buttonStyle(.bordered)
         }
       }
       .padding()
       .navigationTitle("Event Details")
       .navigationBarTitleDisplayMode(.inline)
       .toolbar {
         ToolbarItem(placement: .cancellationAction) {
           Button("Done") { isPresented = false }
         }
       }
     }
     ```
   - Format date: use DateFormatter with `.dateStyle = .full`.
   - Format time range: compute start Date from `event.start` (ms), end Date by adding `event.duration` minutes. Format both with "h:mm a". Show "3:00 PM - 4:00 PM".
   - `@State private var showEditSheet = false` — presents EventEditView as nested sheet.
   - `@State private var showDeleteConfirm = false` — triggers `.confirmationDialog`.
   - Delete confirmation per locked decision:
     ```swift
     .confirmationDialog("Delete this event?", isPresented: $showDeleteConfirm, titleVisibility: .visible) {
       Button("Delete", role: .destructive) {
         Task {
           try? await viewModel.deleteEvent(id: event._id)
           isPresented = false
         }
       }
       Button("Cancel", role: .cancel) {}
     } message: {
       Text("\"\(event.title)\" will be permanently deleted.")
     }
     ```
   - `.sheet(isPresented: $showEditSheet) { EventEditView(...) }`.
